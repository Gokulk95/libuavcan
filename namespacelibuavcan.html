Enumeration Type Documentation

◆ Result

enum libuavcan::Result : std::int_fast8_t
strong

Common return type for functions that report a result. Since libuavcan does not require C++ exceptions this return type is used to signal failures within a function. Negative values and zero are errors. 1 is a nominal result and values > 1 are also considered nominal results.

Enumerator
Success 

Values greater than 0 are success values for libuavcan results.

SuccessPartial 

Some parts of a non-atomic operation completed successfully but other parts failed. This result will only be used where additional information about the failure parts can allow the caller to recover.

SuccessNothing 

The operation didn't do anything but no failures occurred. For example, this would be returned for a read operation that read nothing successfully.

SuccessTimeout 

No errors occurred but the operation did not complete because a timeout period was reached.

BufferFull 

An operation failed because a buffer was full. For some operations this implies that trying again with the same input can be successful.

Failure 

A generic failure.

BadArgument 

One or more parameters provided to a function were invalid.

UnknownInternalError 

The operation experienced an internal inconsistency or an unexpected result from a lower layer.

OutOfMemory 

An operation failed because there was inadequate memory available.

NotFound 

A lookup failed to find anything for the given search parameters.

NotImplemented 

The operation failed because it was not implemented.
